Лаб3: Реализация программного таймера и его API.
На основе шаблона реализовать программный таймер и API для работы в u-моде. Проект должен содержать несколько файлов.

Программа состоит из ассемблерной и C-частей. Ассемблер настраивает PMP, загружает адрес обработчика прерываний в mtvec, 
разрешает прерывания в mstatus и переходит в U-режим на функцию main через mret. Также инициализируется таймер CLINT: включаются 
таймерные прерывания в mie, а mtimecmp устанавливается в максимум.

Обработчик trap_vector сохраняет 32 регистра, вызывает C-функцию handle_trap с аргументами mcause, mepc и указателем на сохранённые регистры, 
затем увеличивает mepc на 4, восстанавливает регистры и выполняет mret.

C-код содержит M-режимные функции hw_timer_init, hw_timer_update, sw_timer_add, sw_timer_is_active. Для вызова из U-режима через ecall 
используется функция syscalls, где в регистре a2 передаётся код операции (1, 2 или 3), а остальные регистры содержат аргументы. 
Для удобства созданы функции-обёртки типа ..._syscall.

Функция handle_trap анализирует cause: при исключении вызывает нужную M-режимную функцию по коду из a2, 
при прерывании проверяет программные таймеры и вызывает callback-функции, которые выводят сообщения в UART. 
Функция main создаёт таймеры через обёртки, затем вызывает wfi для ожидания прерываний и завершается переходом на метку end.
